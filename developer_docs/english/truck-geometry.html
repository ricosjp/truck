Module Guide

Truck Geometry

Practical guidance for working with Truck’s knot vectors, B-splines, and NURBS implementations—the building blocks behind curves, surfaces, and downstream modeling.

Crate: truck-geometry
Precision: f64, cgmath-backed
Highlights: Knot vectors, NURBS curves/surfaces, refinement

1. Geometry foundations

truck-geometry offers spline primitives that satisfy the traits defined in truck-geotrait. You get:

  • Knot vectors – normalized, with helper methods for insertion, removal, and clamping.
  • B-spline / NURBS curves – parameter evaluation, derivatives, reparameterization.
  • NURBS surfaces – tensor-product construction, partial derivatives, normals.

These components are used by modeling (sweeps, lofts), topology (edge trimming), and shape ops (Booleans). Keeping them well-behaved is critical for kernel stability.

2. Knot vectors

Knot vectors partition the parameter axis and dictate basis support.

2.1 Basics & terminology

  • Degree (p) – polynomial degree of the B-spline basis.
  • Order (p + 1) – number of control points influencing a span.
  • Multiplicity – repeated knot values reduce continuity.

Truck stores knots in normalized form (typically [0, 1]). Operators assume non-decreasing order.

2.2 Operations

Common helper methods:

  • insert_knot(t) – increases multiplicity while preserving shape.
  • remove_knot(t) – if error tolerance is met.
  • clamp() – forces start/end multiplicities = degree + 1.

Refinement algorithms (knot insertion/removal, degree elevation) rely on these utilities. Always re-normalize and assert non-decreasing order after manual edits.

3. Curves & evaluation

Curves are represented via control points, optional weights, and a knot vector.

3.1 De Boor & derivatives

Evaluation uses a De Boor-style algorithm. APIs are typically: 

let p = curve.evaluate(t);
let dp = curve.der(t);
let d2 = curve.der2(t);

Derivatives follow the standard recursive basis derivative formulas. Keep parameters within the valid interval ([knots[p], knots[n-p])) or normalize them before evaluating.

3.2 Weights & normalization

NURBS curves multiply control points by weights before performing the basis blend, then divide by the total weight. When editing control points:

  • Use homogeneous coordinates internally ([wx, wy, wz, w]).
  • Expose only affine Point3 values to consumers.
  • Normalize weights to avoid precision issues.

3.3 Parameter domains

Public APIs default to normalized domains. When bridging to truck-geotrait, ensure parameter_range matches the curve’s actual (non-normalized) span so search/project algorithms behave correctly.

4. Surfaces

NURBS surfaces are tensor products of two curve directions.

4.1 Tensor-product construction

Define separate u and v knot vectors plus a grid of control points: 

let surf = NurbsSurface::new(control_net, u_knots, v_knots, degree_u, degree_v);

Evaluation recursively blends along one direction, then the other.

4.2 Operations

  • Partial derivatives – uder, vder, uvder, etc.
  • Normals – computed as uder × vder; Truck normalizes them for integration with ParametricSurface3D.
  • Trimming support – surfaces expose parameter ranges and can be paired with trim curves in truck-topology.

5. Control net management

Control points dictate curve/surface shape. Guidelines:

  • Store them as Point3 (and optionally weights) in row-major order for readability.
  • Use provided iterators/accessors for mutation—direct indexing risks mismatched nets.
  • Local edits propagate only through spans influenced by the edited point, enabling efficient modeling tools.

6. Integration with truck-geotrait

All curves/surfaces implement ParametricCurve / ParametricSurface and their bounded variants. When adding new geometry types:

  • Ensure parameter_range returns proper Bound::Included values.
  • Hook into the search/division traits if you want projection and tessellation to work automatically.
  • Consider using truck-derivers to implement trait forwarding on wrapper structs or enums.

7. Advanced operations

Truck provides building blocks for classic spline editing:

  • Degree elevation – increases degree without changing shape.
  • Knot refinement – insert multiple knots to gain local control.
  • Reparameterization – rescale domains to match modeling requirements.

These routines respect the global tolerance; always propagate tolerance parameters rather than hardcoding epsilons.

8. Summary

truck-geometry delivers the low-level spline machinery the entire kernel depends on:

  • Normalized knot vectors with utility operations.
  • B-spline / NURBS curve and surface evaluation with derivative support.
  • Control net management and refinement helpers.

Mastering these primitives makes it easier to build modeling tools, run Booleans, and integrate with external CAD data.

9. Implementation checklist

  • Domains – keep parameter ranges normalized but expose true bounds via traits.
  • Continuity – verify knot edits preserve non-decreasing order and target continuity.
  • Derivatives – test der/der2/der_mn numerically.
  • Weights – normalize and guard against division by zero.
  • Integration – ensure new primitives implement or derive the required geotrait interfaces.
  • Docs – update this guide / README whenever algorithms or APIs change.
